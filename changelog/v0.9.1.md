A quick fix for the file viewer shipped in 0.9.0: a file you closed could not be opened again. Nothing to migrate.

### 🐛 Fixed

- **Reopening a file after closing its tab.** Opening a file, closing its tab (or pane), then clicking it again in the tree did nothing. Closing a tab tore down its pane and status but forgot the file view behind it, so the tree still believed the file was open and tried to focus a tab that no longer existed. All the close paths now share one teardown step, so a closed file always reopens. ([#18](https://github.com/RizRiyz/bohay/pull/18))
